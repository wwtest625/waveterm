### 36. Shell 注入风险 — `quoteForSingleQuotedShell` 不够安全 ⚠️ 严重 ✅

**文件**: `pkg/aiusechat/tools_writefile.go`

**修复方案**: RPC 直接写入文件，彻底消除 shell 注入

- 删除所有 shell 命令拼装函数：`quoteForSingleQuotedShell`、`runRemoteReadFileCommand`、`runRemoteShellCommandForText`、`runRemoteCommand`
- 新增 RPC 函数：`rpcRemoteWriteFile`（RemoteWriteFileCommand）、`rpcRemoteReadFile`（RemoteStreamFileCommand）、`rpcRemoteDeleteFile`（RemoteFileDeleteCommand）
- `read_text_file` 工具也迁移到 RPC：读全文 + 本地行切片（`strings.Split` + `lines[offset:end]`）
- `usechat.go` 中 diff 预览的文件读取也改为 `rpcRemoteReadFile`

### 38. `edit_text_file` 先读再写存在 TOCTOU 竞态 ⚠️ 中等 ✅

**文件**: `pkg/aiusechat/tools_writefile.go`

**修复方案**: 写入前二次读取 + SHA256 checksum 对比

- 读取文件后计算 `sha256.Sum256` 作为 originalChecksum
- 应用编辑后、写入前，再次读取文件计算 preWriteChecksum
- 如果两个 checksum 不一致，返回错误提示"文件被其他进程修改，请重新读取后重试"
- 新增 `fileChecksum` 辅助函数，新增 `crypto/sha256` import

### 44. `ToolDescLine` 正则匹配 `+N/-N` 过于激进 ⚠️ 轻微 ✅

**文件**: `frontend/app/aipanel/aitooluse.tsx`

**修复方案**: 正则从 `(?<!\w)([+-])(\d+)(?!\w)` 收窄为 `(^|[\s(])([+-])(\d+)(?=[\s,;.)]|$)`

- 要求前导字符为行首、空格或左括号
- 要求后跟字符为空格、逗号、分号、句号、右括号或行尾
- 版本号 `python3.12+` 不再被误匹配
