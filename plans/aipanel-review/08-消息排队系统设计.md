## 一、现状分析

### 已有基础

代码中已存在基本的排队机制：

| 组件 | 文件 | 功能 |
|------|------|------|
| `QueuedSubmission` 类型 | `waveai-model.tsx` L68-73 | 定义排队消息的数据结构（id, text, files, createdAt） |
| `queuedSubmissionsAtom` | `waveai-model.tsx` L99 | 存储排队消息的 jotai atom |
| `enqueueSubmission()` | `waveai-model.tsx` L1597-1611 | 将消息加入队列 |
| `flushQueuedSubmissions()` | `waveai-model.tsx` L1613-1629 | 从队列取出并发送第一条 |
| `isChatBusy()` | `waveai-model.tsx` L1590-1595 | 判断当前是否忙（streaming/loading） |
| 自动 flush | `aipanel.tsx` L1416-1420 | status 变 ready/error 时自动 flush |
| `Queued N` 文字 | `aipanelinput.tsx` L194-195 | 输入区显示排队数量 |

### 缺失能力

1. **不可见**：用户只能看到 "Queued 2"，看不到排了什么消息
2. **不可控**：无法选择"立即发送"（打断当前生成）、无法取消某条排队消息
3. **不可编辑**：排队消息一旦入队无法修改
4. **无引导**：用户不知道可以排队，不知道排队后会发生什么

## 二、交互设计

### 核心场景

```
用户正在看 AI 流式输出，突然想到了一个新问题。
按 Enter 发送 → 消息进入排队序列 → 出现排队卡片
```

### 排队卡片（QueuedMessageCard）

当有排队消息时，在输入框上方显示排队卡片：

```
┌─────────────────────────────────────────────┐
│ ⏳ Queued (2 messages)                       │
│                                              │
│ ┌──────────────────────────────────────────┐ │
│ │ 1. "帮我看看这个 bug"              📎1   │ │
│ │    [▶ Send Now]  [✕ Cancel]              │ │
│ └──────────────────────────────────────────┘ │
│                                              │
│ ┌──────────────────────────────────────────┐ │
│ │ 2. "再查一下日志"                   📎0   │ │
│ │    [▶ Send Now]  [✕ Cancel]              │ │
│ └──────────────────────────────────────────┘ │
│                                              │
│ [Cancel All]                [Send All Later] │
└─────────────────────────────────────────────┘
```

### 三种操作

| 操作 | 按钮 | 行为 |
|------|------|------|
| **排队等待** | 默认行为（已实现） | 消息入队，等当前会话结束后自动发送 |
| **立即发送** | ▶ Send Now | 1. 取消当前生成 2. 等状态变为 ready 3. 发送该消息 |
| **取消排队** | ✕ Cancel | 从队列中移除该消息 |

### 输入框状态变化

当前输入框右侧按钮逻辑：

| 状态 | 现有行为 | 改进后行为 |
|------|----------|-----------|
| Ready | 显示发送按钮 ▶ | 不变 |
| Streaming | 显示停止按钮 ■ | 显示停止按钮 ■ + 如果有排队消息则同时显示排队指示器 |
| 有排队消息 | 无特殊处理 | 输入框上方显示排队卡片，输入框 placeholder 变为 "Add to queue..." |

### 3.2 WaveAIModel 新增方法

```typescript
// waveai-model.tsx — 新增方法

// 取消排队中的某条消息
cancelQueuedSubmission(submissionId: string): void {
    const current = globalStore.get(this.queuedSubmissionsAtom);
    globalStore.set(
        this.queuedSubmissionsAtom,
        current.filter((s) => s.id !== submissionId)
    );
}

// 取消所有排队消息
cancelAllQueuedSubmissions(): void {
    globalStore.set(this.queuedSubmissionsAtom, []);
}

// 立即发送排队中的某条消息（打断当前生成）
async sendQueuedSubmissionNow(submissionId: string): Promise<void> {
    const queued = globalStore.get(this.queuedSubmissionsAtom);
    const target = queued.find((s) => s.id === submissionId);
    if (!target) return;

    // 1. 从队列中移除
    globalStore.set(
        this.queuedSubmissionsAtom,
        queued.filter((s) => s.id !== submissionId)
    );

    // 2. 取消当前生成
    if (this.isChatBusy()) {
        await this.cancelGeneration();
        // 等待状态变为 ready（最多 3 秒）
        const deadline = Date.now() + 3000;
        while (Date.now() < deadline) {
            await new Promise((r) => setTimeout(r, 100));
            if (!this.isChatBusy()) break;
        }
    }

    // 3. 发送该消息
    await this.sendSubmission(target.text, target.files);
}

// 修改现有 enqueueSubmission，增加 status 字段
private enqueueSubmission(text: string, files: DroppedFile[]): void {
    const current = globalStore.get(this.queuedSubmissionsAtom);
    globalStore.set(this.queuedSubmissionsAtom, [
        ...current,
        {
            id: crypto.randomUUID(),
            text,
            files,
            createdAt: Date.now(),
            status: "queued",  // 新增
        },
    ]);
    globalStore.set(this.inputAtom, "");
    this.clearFiles();
    this.clearError();
}
```

### 3.7 自动 flush 逻辑增强

现有逻辑：`status === "ready" || status === "error"` 时自动 flush。

增强：flush 后如果队列还有消息，继续 flush（递归或循环）：

```typescript
// waveai-model.tsx — 增强 flushQueuedSubmissions
async flushQueuedSubmissions(): Promise<void> {
    if (this.isFlushingQueuedSubmission || this.isChatBusy()) {
        return;
    }
    const queued = globalStore.get(this.queuedSubmissionsAtom);
    const next = queued[0];
    if (!next) {
        return;
    }

    // 检查消息状态，跳过已取消的
    if (next.status === "canceling") {
        globalStore.set(this.queuedSubmissionsAtom, queued.slice(1));
        void this.flushQueuedSubmissions(); // 递归处理下一条
        return;
    }

    this.isFlushingQueuedSubmission = true;
    globalStore.set(this.queuedSubmissionsAtom, queued.slice(1));
    try {
        await this.sendSubmission(next.text, next.files);
    } finally {
        this.isFlushingQueuedSubmission = false;
        // 发送完成后，如果队列还有消息，继续 flush
        // 注意：不需要手动调用，因为 status 变 ready 时会触发 useEffect
    }
}
```

## 四、状态流转图

```
用户按 Enter
    │
    ├── isChatBusy() === false
    │       │
    │       └── 直接 sendSubmission()
    │
    └── isChatBusy() === true
            │
            └── enqueueSubmission() → 状态: "queued"
                    │
                    ├── 用户点击 "Send Now"
                    │       │
                    │       ├── cancelGeneration()
                    │       └── sendSubmission()
                    │
                    ├── 用户点击 "Cancel"
                    │       │
                    │       └── cancelQueuedSubmission() → 从队列移除
                    │
                    └── AI 生成完成（status → "ready"）
                            │
                            └── flushQueuedSubmissions() → 自动发送
```

## 五、边界情况

| 场景 | 处理方式 |
|------|----------|
| 排队消息包含文件附件 | 文件随消息一起入队，发送时一起上传 |
| 排队期间用户切换会话 | 切换会话时清空当前队列 |
| "Send Now" 时 cancelGeneration 失败 | 等待最多 3 秒，超时后提示用户重试 |
| 队列中多条消息，第一条 "Send Now" | 只发送第一条，其余保留在队列 |
| AI 生成出错（status = "error"） | 仍然自动 flush 队列（现有逻辑已支持） |
| 排队消息为空文本（只有附件） | 允许入队（与现有行为一致） |
